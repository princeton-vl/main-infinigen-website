## Instructions

In this tutorial, we will show you how to generate a single image with Infinigen, with depth and surface normal annotations:

<AssetPreview

This tutorial does not require a GPU, and assumes all optional compilation steps were skipped.

:::caution Memory Requirements

This tutorial requires at least 20GB of CPU memory.

:::

:::caution Reproducibility

The placement of assets may look different due to package version difference.

:::

To generate this image, run:

```bash
python -m tools.manage_datagen_jobs --num_scenes 1 --specific_seed 46fa3103 \
-o outputs -p full/render_image.num_samples=200  OpaqueSphericalMesher.pixels_per_cube=3 \
animate_cameras.follow_poi_chance=0 --cleanup none \
--pipeline_configs local_16GB monocular blender_gt --config desert fast_terrain_assets \
--wandb disabled --pipeline_overrides LocalScheduleHandler.use_gpu=False
```

The full pipeline should take 20 minutes to 1 hour.

Once the pipeline is finished running, all steps will show as succeeded:

```bash
------------------------------------------------------------

concurrent_max                 : 1

succeeded/blendergt            : 1
succeeded/coarse               : 1
succeeded/fineterrain          : 1
succeeded/populate             : 1
succeeded/short                : 1
------------------------------------------------------------
```

The resulting images and dense annotations will be located in: `outputs/46fa3103/frames_0_0_0001_0`

## Understanding the command

- `manage_datagen_jobs` is the primary python script with which we launch data-generation jobs
- `-p full/render_image.num_samples=200 OpaqueSphericalMesher.pixels_per_cube=3 animate_cameras.follow_poi_chance=0` sets the number of render samples to 200, sets terrain face size to 3 pixels, and disable object-following camera for simplicity
- `--pipeline_configs local_16GB monocular blender_gt` load settings specified in worldgen/tools/pipeline_configs/\*.gin:
  - local_16GB.gin: Use configurations designed for a ~16GB computer.
  - monocular.gin: Rendering one image, at a single timestep.
  - blender_gt.gin: Generate dense annotations using blender's cycles rendering engine.
- `--config desert fast_terrain_assets` load settings specified in worldgen/configs/\*.gin:
  - desert.gin: desert scene
  - fast_terrain_assets.gin: use one terrain tile and run less erosion iterations


:::tip Optional
- If you have a CUDA machine, to have higher resolution and higher quality of terrain, change `LocalScheduleHandler.use_gpu` to `True`, remove `OpaqueSphericalMesher.pixels_per_cube=3` (or smaller) in `-p` and add `high_quality_terrain` in `-g`.
- If you have more time, remove `fast_terrain_assets` to have more variety of terrain and more erosion patterns.
- In this example and next stereo pair example, but not later video example add `-p SphericalMesher.complete_depth_test=0` to skip dense depth test for consistency issue
:::
