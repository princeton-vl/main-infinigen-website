## Instructions

In this tutorial, we will show you how to generate a single image with Infinigen, with depth and surface normal annotations:

This tutorial does not require a GPU, and assumes all optional compilation steps were skipped.

:::caution Memory Requirements

These commands consume in excess of 15GB of memory. Both these issues will be resolved soon.

:::

:::caution Reproducibility

You may obtain a rearranged scene due to platform-specific library versions.

:::

:::caution System

The current example does not yet work for Mac OS, we will add more examples soon.

:::

To generate this image, run:

```bash
cd worldgen
mkdir outputs
../blender/blender --background -noaudio --python generate.py -- --seed 3be70e09 --task coarse -g desert simple --output_folder outputs/helloworld/coarse
../blender/blender --background -noaudio --python generate.py -- --seed 3be70e09 --task populate fine_terrain -g desert simple --input_folder outputs/helloworld/coarse --output_folder outputs/helloworld/fine
../blender/blender --background -noaudio --python generate.py -- --seed 3be70e09 --task render -g desert simple --input_folder outputs/helloworld/fine --output_folder output/helloworld/frames
../blender/blender --background -noaudio --python generate.py -- --seed 3be70e09 --task render -g desert simple --input_folder outputs/helloworld/fine --output_folder output/helloworld/frames -p render.render_image_func=@flat/render_image
```

For GPU-based rendering, change `LocalScheduleHandler.use_gpu=False` to `True`

The full pipeline should take 20 minutes to 1 hour.

Once the pipeline is finished running, all steps will show as succeeded:

```bash
------------------------------------------------------------
<output_folder> 06/15 08:09AM -> 06/15 08:16AM
concurrent_max                 : 1
disk_usage                     : 0.86
succeeded/all                  : 5
succeeded/blendergt            : 1
succeeded/coarse               : 1
succeeded/fineterrain          : 1
succeeded/populate             : 1
succeeded/short                : 1
------------------------------------------------------------
```

The resulting images and dense annotations will be located in: `outputs/46fa3103/frames_0_0_0001_0`

## Understanding the command

- `manage_datagen_jobs` is the primary python script with which we launch data-generation jobs.
- `--specific_seed 46fa3103` sets the initial random seed.
- `-p full/render_image.num_samples=200 OpaqueSphericalMesher.pixels_per_cube=3 animate_cameras.follow_poi_chance=0` sets the number of render samples to 200, sets terrain face size to 3 pixels, and disables the object-following camera for simplicity.
- `--pipeline_configs local_16GB monocular blender_gt` loads settings specified in worldgen/tools/pipeline_configs/\*.gin:
  - local_16GB.gin: Use configurations designed for a ~16GB memory computer.
  - monocular.gin: Render one image, at a single timestep.
  - blender_gt.gin: Generate dense annotations using blender's cycles rendering engine.
- `--config desert fast_terrain_assets` loads settings specified in worldgen/configs/\*.gin:
  - desert.gin: desert scene
  - fast_terrain_assets.gin: use one terrain tile and run fewer erosion iterations.

:::tip Optional

- If you have a CUDA-enabled graphics card and would like higher resolution and higher quality terrain, change `LocalScheduleHandler.use_gpu` to `True`, remove `OpaqueSphericalMesher.pixels_per_cube=3` (or smaller) in `-p` and add `high_quality_terrain` in `-g`.
- Remove `fast_terrain_assets` to have more variety of terrain and more erosion patterns.
- In this example add `-p SphericalMesher.complete_depth_test=0` to skip dense depth test for consistency issue.

:::
